# Desktop — Proxy Access Client (PyQt6)

Десктоп-клиент для активации ключа, отображения назначенной VM и мониторинга подключения в реальном времени. Интерфейс в стиле Happ: левый сайдбар, центральный список серверов, большой круглый индикатор статуса справа.

## Возможности

- **Активация ключа** через модальный диалог (`POST /api/activate-key`) с проверкой доступности backend
- **Автоматическое обновление access-токена** при истечении (через refresh-токен внутри `ApiClient`)
- **Сохранение последней назначенной VM** — карточка сервера видна в списке даже после отключения, чтобы быстро переподключиться
- **Кликабельный круглый индикатор** — `POST /api/connect` / `POST /api/disconnect` в один клик
- **Realtime-мониторинг** через WebSocket `/ws/connection-status` (4 статуса: `connected` / `disconnected` / `no_free_vms` / `error`)
- **Локальное хранение сессии** в `%APPDATA%\ProxyAccess\session.json` — приложение запоминает токены между запусками
- **Тёмная тема** через QSS

## Требования

- **Python 3.11+** (рекомендуется 3.12)
- **Windows 10/11** (тестировалось на Windows)
- **Запущенный backend** (`http://localhost:8000` по умолчанию)

> На Linux/macOS должно работать тоже — PyQt6 кроссплатформенный, но конкретный QSS отлаживался под Windows.

## Установка и запуск

### Быстрый старт (Windows PowerShell)

```powershell
cd Desktop
.\run.ps1
```

Скрипт сам создаст `.venv`, поставит зависимости и запустит приложение.

### Вручную

```powershell
cd Desktop
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
.\.venv\Scripts\python.exe run.py
```

## Конфигурация

Создайте файл `Desktop/.env` (необязательно — есть дефолты):

```env
# Адрес backend API
PROXY_ACCESS_API_URL=http://localhost:8000
```

Файл загружается автоматически при старте приложения через `python-dotenv` (см. `proxy_access/config.py:10-11`).

| Переменная | Default | Где используется |
|---|---|---|
| `PROXY_ACCESS_API_URL` | `http://127.0.0.1:8000` | Базовый URL для всех HTTP и WebSocket запросов |

## Сценарий использования

1. **Регистрация** — на сайте `http://localhost:5173/register`, ключ приходит на email
2. **Запуск Desktop** — `Desktop\run.ps1`
3. **Первый запуск** — автоматически открывается диалог «Активация ключа»
4. **Вставка ключа** — `Ctrl+V` или кнопка «Вставить», затем «Активировать»
5. **После активации** — открывается главное окно:
   - Слева: сайдбар с кнопками `+` (новый ключ), серверы, выход
   - Центр: карточка с назначенной VM (`VM #N · SOCKS5 · 10.0.0.X:1080`)
   - Справа: большой круг — индикатор статуса + кликабельная кнопка ВКЛ/ВЫКЛ
6. **Подключение/отключение** — клик по большому кругу:
   - Серый круг (disconnected) → клик → берём VM из пула
   - Зелёный круг (connected) → клик → освобождаем VM
   - При этом карточка с VM **остаётся видна** для быстрого переподключения
7. **WebSocket** автоматически обновляет статус каждые 3 секунды без действий пользователя

## Состояния круглого индикатора

| Цвет | Статус (WS) | Лейбл | Кликабельность |
|---|---|---|---|
| зелёный | `connected` | ПОДКЛЮЧЁН | да — отключение |
| серый | `disconnected` | НЕ ПОДКЛЮЧЁН | да — подключение |
| оранжевый | `no_free_vms` | НЕТ СВОБОДНЫХ | да — retry |
| красный | `error` | ОШИБКА | нет (только индикатор) |
| фиолетовый | `loading` | ОЖИДАНИЕ… | нет (запрос идёт) |

Неизвестный статус трактуется как `error` (fail-safe — не покажем «зелёный» при поломке).

## Локальное хранение

Сессия сохраняется в:

```
Windows:  %APPDATA%\ProxyAccess\session.json
Other:    ~/ProxyAccess/session.json   (fallback)
```

Структура:

```json
{
  "api_url": "http://localhost:8000",
  "user_id": 1,
  "access_token": "eyJ...",
  "refresh_token": "eyJ...",
  "proxy": { "host": "10.0.0.5", "port": 1080, "protocol": "socks5" },
  "vm_id": 5,
  "activated_proxy": { "host": "10.0.0.5", "port": 1080, "protocol": "socks5" },
  "activated_vm_id": 5
}
```

- `proxy` / `vm_id` — **текущее активное подключение**. NULL при disconnect, может отличаться от `activated_*`, если `/api/connect` выдал другую VM (потому что активированная была занята).
- `activated_proxy` / `activated_vm_id` — **сервер, выданный при последней активации ключа**. Записывается один раз при успехе `/api/activate-key` 

Удалить сессию принудительно (равноценно «Сбросить» в UI):

```powershell
Remove-Item "$env:APPDATA\ProxyAccess\session.json"
```

## Сборка одиночного `.exe`

Через PyInstaller:

```powershell
cd Desktop
.\.venv\Scripts\Activate.ps1
pip install pyinstaller
pyinstaller --onefile --windowed --name ProxyAccess `
    --add-data "proxy_access\ui\styles.qss;proxy_access\ui" `
    run.py
```

Готовый `.exe` появится в `dist\ProxyAccess.exe` (≈40 MB).

- `--onefile` — всё в одном .exe (медленный старт, но переносимо)
- `--windowed` — без чёрного консольного окна
- `--add-data` — встроить QSS в бандл (иначе тема не подхватится)

> При запуске собранного `.exe` `__file__` указывает в распакованный временный каталог. Если `proxy_access/main.py` грузит QSS относительным путём — проверьте, что он работает после PyInstaller-сборки.

## Troubleshooting

| Симптом | Причина | Решение |
|---|---|---|
| `module 'websocket' has no attribute 'create_connection'` | Запуск из системного Python без `websocket-client`, либо конфликт со старым пакетом `websocket` (без `-client`) | Использовать `.\.venv\Scripts\python.exe`; если не помогло — `pip uninstall -y websocket && pip install websocket-client` |
| `No module named 'PyQt6.sip'` | Версии `PyQt6` / `PyQt6-Qt6` / `PyQt6-sip` несовместимы | `pip install --force-reinstall -r requirements.txt` (версии в `requirements.txt` зафиксированы для совместимости) |
| Внутри круга / в кнопке рендерятся «квадраты» вместо символа | Шрифт Segoe UI не содержит нужного глифа (например, `⏻` U+23FB) | В текущей версии UI символ убран. Если квадраты остались на других кнопках — заменить на надёжные глифы (`+`, `●`, `mdi-*` шрифт) |
| `Не удалось связаться с сервером: http://localhost:8000/` | Backend не запущен | `docker compose up -d backend postgres redis` |
| После активации сразу показывается «НЕТ СВОБОДНЫХ» | В пуле нет свободных VM, ключ остался валидным | Админ должен добавить VM через `/admin`, потом нажать большой круг — повтор активации |
| Статус «ОШИБКА — Мониторинг переподключается…» | WebSocket разорвался / истёк access-токен / backend упал | Подождать несколько секунд — `StatusWorker` авто-переподключается каждые 3 сек |
| Приложение «зависло» при закрытии на ~2 секунды | Дожидается остановки фоновых потоков (`status_worker.wait(2000)`) | Нормально — это плановый graceful shutdown |
| После смены сети меняется IP, но клиент это не видит | Адрес VM приходит из backend, клиент не делает локальный probe | By design — клиент полагается на серверный статус |
